What is Icos Search?

Icos Search is a privacy-focused meta search engine that acts as a protective barrier between you and Google. As a meta search engine, we don't have our own search index - instead, we fetch Google's search results for you while keeping your identity completely anonymous. You get the same high-quality search results from Google without being tracked, profiled, or targeted with ads.

"Search the web privately without sacrificing result quality."
— Icos Search Mission

Advantages Over Google

  • Complete Anonymity: Google never sees your IP address, browser fingerprint, or personal information
  • No Tracking: Zero search history logging, no user profiling, no behavioral tracking
  • Ad-Free Experience: All Google ads and sponsored content automatically removed
  • Script Protection: Tracking JavaScript and analytics code filtered out
  • Clean Interface: No distractions, just pure search results
  • Fast Loading: Faster page loads without ads and tracking scripts
  • Privacy Controls: Customizable settings without account registration

    Before showing you search results, Icos automatically strips out Google's ads, tracking scripts, and data collection mechanisms that would normally follow you around the web.
    What we remove: All Google ads, tracking pixels, analytics scripts, and data collection code are filtered out before results reach your browser.

    Benefits: • No targeted advertising • Faster page loading • No cross-site tracking • Clean, distraction-free results
    JavaScript & Script Blocking
    Icos removes harmful JavaScript and tracking scripts from search results, preventing websites from running code that could track or fingerprint your browser.
    Protection level: We filter out tracking scripts, analytics code, and fingerprinting attempts while preserving essential functionality needed for search results.

    Advanced Cryptographic Security

    Icos Search implements robust cryptographic techniques that go far beyond standard HTTPS encryption. Our custom SecureURLShield (SX) system provides strong encryption protection for search queries and URLs within the Icos search system.

    SecureURLShield (SX) Encryption System

    Search result URLs are protected with our proprietary SX encryption within the Icos system - making it impossible for anyone to see what you searched for by looking at URLs in your browser history or network logs.

    AES-256-CBC Encryption
    We use industry-standard Advanced Encryption Standard with 256-bit keys in Cipher Block Chaining mode - a proven and highly secure encryption method.
    Technical Implementation: • AES-256 (Advanced Encryption Standard, 256-bit key) • CBC Mode (Cipher Block Chaining for added security) • Each URL gets its own unique encryption key • Cryptographically secure random number generation

    Why This Matters: • Industry-standard encryption trusted worldwide • Extremely difficult to crack with current technology • Each encrypted URL is completely unique • No two identical URLs produce the same encrypted output
    PBKDF2 Key Derivation
    We generate unique encryption keys for every URL using PBKDF2 (Password-Based Key Derivation Function 2) with 100,000 iterations and SHA-256 hashing.
    Key Generation Process: • PBKDF2 with 100,000 iterations (industry best practice) • SHA-256 cryptographic hash function • Combines master key + URL hash + random salt • Produces unique 32-byte encryption key per URL

    Security Benefits: • Prevents rainbow table attacks • Each URL has completely different encryption • Keys are never reused or predictable • Computationally expensive to brute force
    Cryptographic Salt & IV
    Every encrypted URL includes a unique 16-byte salt and 16-byte initialization vector (IV) generated using cryptographically secure random number generation.
    Randomization Components: • 16-byte random salt per URL (prevents pattern recognition) • 16-byte random IV per URL (ensures encryption uniqueness) • Generated using secrets.token_bytes() - cryptographically secure • Never reused across different URLs

    Why Random Salt/IV Matters: • Same URL encrypted twice = completely different output • Prevents frequency analysis attacks • Makes pattern recognition impossible • Ensures forward secrecy
    Custom Base64 Encoding
    After encryption, URLs are encoded using our custom Base64 alphabet that completely obscures the encrypted data and makes it unrecognizable.
    Custom Alphabet System: • Uses a proprietary character mapping instead of standard Base64 • Custom alphabet sequence kept private for security • Makes encrypted URLs completely unrecognizable • Even if someone knows it's Base64, they can't decode it without our custom alphabet

    Obfuscation Benefits: • Encrypted URLs look like random gibberish • No recognizable Base64 patterns • Additional layer of security through obscurity • Custom encoding unique to Icos Search
    SHA-256 Integrity Verification
    Every encrypted URL includes a SHA-256 hash for integrity verification, ensuring the URL hasn't been tampered with during transmission.
    Integrity Protection: • SHA-256 hash of original URL embedded in encryption • Verified during decryption process • Detects any tampering or corruption • Ensures URL authenticity

    Anti-Tampering Benefits: • Prevents man-in-the-middle attacks • Detects data corruption instantly • Ensures you visit the intended website • Cryptographic proof of URL integrity
    SX Prefix System
    Our encrypted URLs use the "SX" prefix (SecureX) instead of standard URL formats, making them instantly recognizable as Icos-protected links.
    SX Identification System: • All encrypted URLs start with "SX" prefix • Replaces Google's "gAAA" system with our own • Instantly identifies Icos-encrypted content • Example: SXZyxwvut... (completely obfuscated URL)

    System Benefits: • Clear identification of protected URLs • No confusion with standard web addresses • Proprietary to Icos Search system • Enables automatic detection and handling
    "Search result URLs within Icos are protected by robust encryption standards that ensure your privacy and security."
    — Icos Cryptographic Security

    SX System & Vortex System: Search Privacy Architecture

    SX System: Cryptographic URL Shield
    SX (SecureX) is our comprehensive cryptographic system that combines all advanced encryption techniques into one unified protection framework for every URL and search query.
    SX System Components: • AES-256-CBC encryption (industry-standard) • PBKDF2 key derivation with 100,000 iterations • Cryptographic salt & IV generation • Custom Base64 encoding with unique alphabet • SHA-256 integrity verification • SX prefix identification system

    Real-World Privacy Impact:

    Vortex is our encrypted preference storage system that securely manages all your settings and customizations using the same cryptographic standards as the SX System.
    Vortex-Managed Preferences: • Theme settings (Light/Dark/System) • Safe search filtering (On/Off) • Link behavior (Same tab/New tab) • Language preferences • Search result filtering options • Interface customizations • AI sidebar toggle settings

    Encryption Protection: • All preferences encrypted using SX cryptographic standards • Local browser storage with strong encryption • No server-side preference tracking or logging • Preferences remain private even from network analysis • Zero correlation possible between user and settings

    How Icos Works

    Icos Search operates as a privacy-preserving proxy between you and Google Search, using proven cryptographic techniques and security protocols to ensure your anonymity while delivering high-quality search results.

    Proxy Architecture
    When you search with Icos, your request goes to our servers first. We then query Google on your behalf, strip out tracking elements, and deliver clean results back to you.
    Privacy Process: 1. Your search query is sent over HTTPS to Icos servers 2. Request is anonymized using IP masking techniques 3. Icos forwards anonymized query to Google 4. Google returns search results to Icos 5. Results are filtered using content parsing algorithms 6. Clean results delivered back over encrypted connection

    Cryptographic Protection: • TLS 1.3 encryption for all communications • Forward secrecy to protect past sessions • Certificate pinning for secure connections • No persistent session tokens
    Security Techniques
    We implement industry-standard security practices to protect your privacy and ensure secure communications throughout the search process.
    Security Implementations: • HTTPS/TLS encryption for all data transmission • Content Security Policy (CSP) headers • XSS protection and input sanitization • Regular security headers implementation • Memory-safe request handling

    Privacy Techniques: • Request header anonymization • User-Agent randomization • No logging of search queries or user data • Automatic session cleanup
    Anonymous Session Management
    Your preferences are stored locally using browser storage APIs without requiring accounts or collecting personal information.
    Local Storage Implementation: • Browser localStorage for preferences • No server-side user tracking • Encrypted preference storage • Client-side configuration management

    Technical Benefits: • No authentication required • Zero personal data collection • Portable settings across devices • Complete user control over data
